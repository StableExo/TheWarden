## Contributing to Documentation

When adding new documentation:
1. Place detailed guides and architecture docs in the main `docs/` directory
2. Place summaries and reports in `docs/reports/`
3. Place deployment-related docs in `docs/deployment/`
4. Place integration guides in `docs/integration/`
5. Place security audits/reviews in `docs/security/`
6. Update this README.md to include links to new documentation
